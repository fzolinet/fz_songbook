The idea came from Linkesoft Songbook

This module you can use two mode

  • 1. [songbook src="path-to-file/filename.pro" /]
    The filter read the songbook file and shows it just like a songbook in the body of node.
  • 2. [songbook]...songbook text... [/songbook]
    The filter module shows the text between the tags like a songbook text in the node.

In the songbook file or text you can mix the lyrics of song and the accords or any other informations, tabs, notes, etc.

About the songbook text / file

The header of songbook text

In the header you can use tags

  • {title:....} or {t:....} - the title of song
  • {subtitle: ...} or {st: ...} - the subtitle of song, for example the name of composer, lyricist
  • {time:...} - the time of song
  • {key:...} - the key of the song
  • {tag:....} - the tag can be anything, for example rock, Something Blues Band...
  • {musicpath:.....} - the place of sound material. It can:
    • http://, https://, ftp:// - from the internet, for example {musicpath:https://www.youtube.com/watch?v=WHtWs4wiFCs}.
    • public:// - from Drupal filesystem public folder. for example: {musicpath:public://songbook/something.mp3}
    • file:// - From settings of fz_songbook, tipically public://, {musicpath:file://songbook/something.mp3}
    • something /folder/filename.mp3 - From root of Drupal. For example: {musicpath:/songbook/something.mp3}
    • If SmplPhotoalbum installed the path will be a subfolder of Smplphotoalbum root folder.
      • /path-to-smplphotoalbum/v.php?i=/Folder/filename.mp3 will shows only the filename, but the link helps the play this song. For Example: {musicpath:/sites/all/modules/smplphotoalbum/v.php?i=/Folder/filename.mp3}
      • /folder/filename.mp3 - shows only filename but the link help to play the song with smplphotoalbum. For example: {musicpath:/folder/filename.mp3}

The body of the songbook text

The text could be:

  • [ChordName] - it shows above the text the right chord sign in other color. For Example The [Em7]love is beautyful
  • {c:.....} - is a comment. {c:Refrain}
  • {soc} .......{eoc} - start of comment , end of comment.
  • {sot} .......{eot} - start of tabulature, end of tabulature
  • {sog} .......{eog} - start of chord grid, end of chord grid
